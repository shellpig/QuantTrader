# shellpig 版規格未涵蓋的資料項目

> 建立日期：2026-04-25
> 對應規格：`量化交易系統規格書_shellpig版.md`（V1.2）

## 目前已涵蓋的（提醒對照用）

| 類別 | 欄位 / 內容 | 取用 API |
|---|---|---|
| 日K（未復權） | OHLCV + symbol（7 欄 `STANDARD_COLUMNS`） | `storage.load_daily(symbol)` |
| 分K | 同上，freq ∈ {1, 5, 15, 30, 60} 分鐘 | `storage.load_minute(symbol)` |
| 前復權日K | 同 7 欄，OHLC 已套 `adj_factor` | `storage.load_adjusted(symbol)` |
| 除權息事件 | date / cash_dividend / stock_dividend / symbol | `FinMindFetcher.fetch_dividends(symbol)`（不落地） |
| 元資料 | (symbol, freq, source, first_date, last_date, row_count, updated_at) | `DuckDBMeta.list_all()` |
| 品質報告 | negative/zero price、extreme change、zero volume、suspensions | `DataCleaner.clean(df, symbol)` 第二回傳值 |

## 如果之後要加，要動哪些檔案

以「想加月營收」為例：

1. `src/data/fetcher.py`
   - 在 `IDataFetcher` 加 `fetch_monthly_revenue(symbol, start, end)` 抽象方法
   - 在 `FinMindFetcher` 實作（`dataset="TaiwanStockMonthRevenue"`）
   - 在 `YFinanceFetcher` 視情況留 `NotImplementedError` 或從其他來源實作

2. `src/core/constants.py`
   - 加新的 `MONTHLY_REVENUE_COLUMNS = ["date", "revenue", "yoy", "mom", "symbol"]`

3. `src/data/storage.py`
   - 新增 `_revenue_path()` 例如 `data/raw/tw/{symbol}/monthly_revenue.parquet`
   - 新增 `save_monthly_revenue` / `load_monthly_revenue`
   - 在 `_normalize_market_df` 之外另寫一個 `_normalize_revenue_df`（schema 不同）

4. `src/data/cleaner.py`
   - 視需求決定是否做品質檢核（營收資料品質通常較高，可能不需要 L1/L2）

5. `src/data/maintenance.py`
   - 在 `rebuild_symbol` 加 `fetch + save_monthly_revenue` 步驟
   - 元資料用新的 `freq="monthly_revenue"` 寫進 `DuckDBMeta`

6. 測試
   - `tests/fixtures/revenue_2330.csv` 固定資料
   - `tests/test_fetcher.py` 加 `fetch_monthly_revenue` 單元/整合測試
   - `tests/test_storage.py` 加 roundtrip 測試
   - `tests/test_maintenance.py` 加新流程測試

7. 文件
   - 在 `量化交易系統規格書_shellpig版.md` 加新章節
   - 在 `開發設計方針.md` 標示新增的 phase / sub-phase
   - 在 `測試指南.md` 加驗收標準
   - **更新本文件**，把該項目從「未涵蓋」移除

---

## 紀律

> **不要在沒擴規格書的情況下偷偷加新 fetcher。**
> shellpig 版的核心精神是「先把回測引擎與風控做扎實，資料源僅做最小可行集」，
> 任何資料擴充都應該先有「為什麼策略需要它」的說明，再回到 Phase 1 的管線補。
