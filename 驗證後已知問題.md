### 🚨 排版可讀性硬性要求（重要）

中 / 右欄寬度從 360px 縮到約 314px 後，**所有 panel 內元素都必須通過下列檢查**，否則視為本輪未完成：

#### A. 不允許的視覺缺陷

- ❌ **文字超出 panel 邊框**（橫向溢出）
- ❌ **數字 / 單位被截斷**（例如「成交量」變「成交...」、「+39,219 張」變「+39,...」）
- ❌ **Badge 文字被擠成多行**（如 round-3 #11 一開始的「未/成」直排狀況）
- ❌ **表格欄位寬度衝突導致換行錯亂**（K 線型態 / 籌碼明細表）
- ❌ **大字體（如關鍵價位數字）字寬大於 panel 寬度導致溢出**
- ❌ **HelpTooltip 的 `?` icon 與相鄰文字過於擁擠看不出歸屬**

#### B. 必要的對應措施

| 問題場景 | 處置 |
|---|---|
| 中英混排造成長 word 不換行 | `break-words` 或 `overflow-wrap: anywhere` |
| 數字欄位（如「賣超 39,219 張」）橫向太長 | 表格 `td` 用 `whitespace-nowrap`，整列允許橫向 scroll；或縮 `text-sm` → `text-xs`；或省略「張」字（已有單位欄） |
| panel 標題太擠 | 標題 `<h3>` 用 `text-xs` 而非 `text-sm`，或省略次要文字（如「近 60 日」「近 5 日」） |
| 多欄並排（如關鍵價位 label + 數字）對齊問題 | 統一用 `flex justify-between items-baseline` 而非 `grid` |
| K 線型態表格三欄擠不下 | 「狀態」欄 fixed 64px、「型態」欄 auto、「說明」欄 `flex-1 min-w-0 break-words` |
| 多週期 trend badge 文字（「多頭」「空頭」）變短 | 已是 2 字無壓力；strength 欄位數字長則 mono |
| AI 劇本三情境卡 grid-cols-3 在窄欄擠 | 改 `grid-cols-1` 或 `grid-cols-2` 視寬度決定 |

#### C. 文字尺寸層級規範（本輪統一）

| 用途 | class | 約定 |
|---|---|---|
| Panel 標題 | `text-sm font-semibold` | 不再用 `text-base` 以上 |
| Panel 副標 / 計量單位 | `text-xs text-slate-500` | 「近 60 日」「近 5 日」這類 |
| 主數值（價格 / 量） | `text-base [font-family:var(--font-mono)]` | 從 round-4 的 `text-2xl` 縮到 base |
| 一般說明 | `text-sm` | 不要 `text-base` |
| Tooltip / Help icon | `h-3.5 w-3.5` | 不放大 |

#### D. 驗收時必看的位置

實作完成後 **逐一在瀏覽器確認**：

1. 技術分析總覽：每一列 label + 值能完整顯示，「多頭排列 (5>20>60)」這類較長 status 字串不被截

5. 多週期表格：日 K / 週 K / 月 K 三列、強度欄位不擠
6. 操作觀察段落：完整中文段落自然換行，不出框、不被截斷

## [P10-C-bug] Phase 10-C 資料管理頁 — 手動測試發現的 2 個 bug — ✅ 全部修正、已驗證

> 2026-05-15 使用者實機切換 TW / US market 時發現，10-C 列表行為有兩個未在自動測試覆蓋的問題。**2026-05-15 修正完成**。

### Bug 1：切換 market 時 React duplicate key + 跨 market 資料殘留 🔴

**現象**
- 切換台股 / 美股時，瀏覽器 Console 報 `Encountered two children with the same key, '0050'`
- 視覺上選台股時可能殘留美股列，反之亦然

**根因**
- `data_meta` 的 PRIMARY KEY 是 `(market, symbol, freq)`（[storage.py:558](src/data/storage.py:558)）。一檔台股 `0050` 通常有 `daily / minute / institutional / margin` 四列。
- [data_service.py:64-87](src/services/data_service.py:64) `list_symbols()` 只做 `market` filter，**未對 symbol 去重**，整批 N 列同 symbol 全送前端。
- [useDataList.ts:14-16](web/src/lib/hooks/useDataList.ts:14) 取 `row.symbol` 沒去重 → [DataTable.tsx:47](web/src/components/data/DataTable.tsx:47) `key={row.symbol}` 重複。
- React 在 keys 重複時 reconciliation 行為不穩（官方訊息：「could be duplicated and/or omitted」），切 market 時舊 DOM 節點可能被誤重用 → 看起來像跨 market 殘留。

**功能影響**
- 動作欄「更新 / 刪除」按鈕送出的 API 仍帶正確 `row.market` + `row.symbol`，**後端方向正確**；但 UI 列數錯、視覺資料錯。

**處理方式（採用方案 A：後端去重）**
- 修 [src/services/data_service.py:64](src/services/data_service.py:64) `list_symbols`：在 `df = df[df["market"] == normalized_market]` 之後加
  ```python
  df = df.drop_duplicates(subset=["symbol"], keep="first")
  ```
- 理由：API 語意「列出該 market 有資料的標的」本來就應該是 per-symbol，目前粒度錯了；修一個地方所有 client 都受益。
- （備案 B：前端 `Array.from(new Set(...))` 去重 — 不採用，需要每個 caller 各做一次容易遺漏。）

**驗證計畫**
- 新增 `tests/test_api/test_data_api.py` 一個 case：在 `data_meta` 插入同 (tw, 0050) 多 freq → `GET /api/data/symbols?market=tw` 只回 1 筆。
- 手動：切 TW ↔ US ↔ TW 數次，Console 0 warning、列表內容無殘留。
