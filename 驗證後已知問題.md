### D. 預設 K 棒視野 = 6 個月

- **現況**：[candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx) 沒有設定預設可視範圍，使用 lightweight-charts 預設（顯示全部資料）
- **使用者決定（D-Q1）**：日 K 預設**顯示最近 6 個月**（約 120 個交易日）
- **實作**：在 chart 載入資料後呼叫
  ```ts
  chart.timeScale().setVisibleLogicalRange({
    from: Math.max(0, bars.length - 120),
    to: bars.length - 1,
  });
  ```
- **注意**：interval 切換時（日 / 週 / 月 / 分 K）需個別決定預設視野
  - 日 K：120 bars（6 個月）
  - 週 K：~26 bars（6 個月）
  - 月 K：~12 bars（1 年）
  - 分 K：~120 bars（半天）
- **影響**：[candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx) `useEffect` data load 區塊

### E. 壓力 / 支撐線畫在 K 線圖上 + 移除前日收盤虛線

- **現況**：
  - [candlestick-chart.tsx:308-315](web/src/components/dashboard/candlestick-chart.tsx:308) candlestick series 沒設 `priceLineVisible: false`，所以**自動畫出「最後收盤價」橫貫虛線**（使用者觀察到的「前日收盤虛線」）
  - 沒有畫壓力 / 支撐線
- **使用者決定**：
  - **E-Q1**：S/R 線**接受橫貫整圖**（lightweight-charts 的 `createPriceLine` 本身就是橫貫，舊 Streamlit 雖非橫貫但接受此實作差異）
  - **附帶**：**移除**目前的「前日收盤橫貫虛線」（這個是 lightweight-charts 預設行為，給 candle series 設 `priceLineVisible: false` 即可）
- **資料來源**：後端 `payload.technical.resistance_levels: PriceLevel[]` / `support_levels: PriceLevel[]` 已有資料（[technical_summary.py](src/analysis/technical_summary.py) 已實作）
- **實作**：
  - candle series 加 `priceLineVisible: false`
  - 接受新 props `resistanceLevels: PriceLevel[]` / `supportLevels: PriceLevel[]` 從 `dashboard-page-client.tsx` 傳入
  - 對每個 level 呼叫
    ```ts
    candle.createPriceLine({
      price: level.value,
      color: '#ef4444',  // 紅虛線 = 壓力
      lineStyle: LineStyle.Dashed,
      lineWidth: 1,
      axisLabelVisible: true,
      title: '壓力',  // 或 level.label
    });
    ```
    支撐用 `#22c55e` 綠色，title 「支撐」
  - 用 ref 追蹤 priceLine 實例，在 symbol / market 切換時 `candle.removePriceLine()` 清掉舊的
- **顏色說明**：S/R 顏色獨立於市場（紅綠不隨台/美股反轉），因為「壓力上方紅、支撐下方綠」是視覺語意而非漲跌方向
- **影響**：
  - [candlestick-chart.tsx](web/src/components/dashboard/candlestick-chart.tsx) 加 props / createPriceLine / 移除預設 priceLine
  - [dashboard-page-client.tsx](web/src/components/dashboard/dashboard-page-client.tsx) 傳遞 S/R levels 給 chart

### 風險 / 注意事項（round 4）

| # | 風險 | 緩解 |
|---|---|---|
| A | `PATTERN_DETAILS` 字典若 backend 新增型態名稱，前端會找不到 key → tooltip 空 | tooltip 內容 fallback：`PATTERN_DETAILS[name] ?? '（暫無詳細說明）'` |

| E | resistance/support 顏色固定紅綠，與台股/美股漲跌色語意可能混淆 | 在 round 4 接受此設計；若後續使用者反映再改 |
| F | `w-32` 後 nav item 文字可能緊貼 icon 不易閱讀 | 改完手動驗證；必要時 nav item padding 微調 |
