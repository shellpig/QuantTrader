## 0. 背景與目標

### 0.1 動機

Phase 1-7 完成了資料管道、回測引擎、策略擴充、參數掃描、Walk-Forward Analysis。但目前系統缺少**個股即時綜合分析**能力——使用者想對單一標的快速掌握技術面、籌碼面、AI 判讀的全貌，而不是每次都跑回測。

### 0.2 參考基準

使用者提供的截圖為一個 12 區塊的個股分析儀表板（3583 辛耘），包含：

| # | 區塊 | 核心內容 |
|:---|:---|:---|
| 1 | 頂部行情條 | 即時價 / 漲跌 / 成交量 / 內外盤比 |
| 2 | 日 K 線圖 | K 線 + MA 均線 + 成交量 + 支撐壓力標註 |
| 3 | 技術分析總覽 | 趨勢方向 / MA 狀態 / KD / MACD / 量價關係（文字判讀） |
| 4 | 產業概況 / 公司概況 | 產業地位、營運強度（文字描述） |
| 5 | 籌碼分析 | 外資 / 投信 / 自營商近 N 日買賣超 |
| 6 | 主力出貨警示 | 籌碼集中度 / 穩定度（儀表圖） |
| 7 | 短線勝率 | 近 10 日上漲比例（圓弧儀表） |
| 8 | 關鍵價位 | 自動計算壓力區 / 支撐區 |
| 9 | 量價結構分析 | 量價背離 / 均線乖離 / 主力行為 / 操作觀察（文字） |
| 10 | 內外盤結構 | 買盤 / 賣壓比例 |
| 11 | K 線型態表 + 型態分析 | candlestick pattern 辨識 / W 底 M 頭偵測 |
| 12 | 多週期分析 + 隔日操作劇本 | 日 / 週 / 月趨勢 + 三情境操作建議 |

> 參考圖中的「短線勝率」與「內外盤」為視覺/產品參考名稱；Phase 8 實作需改用「短線綜合分數」與「買賣力道估算」，避免把估算指標包裝成精準統計或交易所原始欄位。

### 0.3 設計決策

| 決策 | 選擇 | 理由 |
|:---|:---|:---|
| UI 框架 | **Streamlit（維持現有）** | 不重寫前端，工程量可控 |
| 排版策略 | **Tab 分頁 + 滾動式**（路線 1） | Streamlit 無法單屏塞 12 面板；tab 分頁可讀性更好 |
| 即時行情 | **TWSE MIS API** | 免費、免 key、盤中即時（約 5-15 秒延遲）；用 `ex_ch=tse_` / `otc_` 支援上市與上櫃 |
| 籌碼資料 | **FinMind 免費層** | 三大法人買賣超可用；券商分點降級為法人替代 |
| 刷新機制 | **手動按鈕**（不做 auto-refresh） | Streamlit auto-rerun 會重跑整頁分析，體驗差 |
| K 線型態 | **`pandas_ta.cdl_pattern(name=...)` + `cdl_doji` + 自定規則** | 技術確認：`cdl_pattern` 存在，`CDL_PATTERN_NAMES` 含 62 種型態（含 hammer/engulfing/morningstar 等所需型態） |
| 專案定位 | **研究、回測、盤中觀察，不接實盤下單** | 加入即時報價與操作劇本後，仍不可讓 UI 語意變成實盤建議 |

### 0.4 資料可信度等級

參考圖中部分區塊可由行情與歷史 K 線精準計算，部分只能用免費資料近似。Phase 8 UI 與 AI prompt 必須保留資料可信度標示，避免畫面看起來比資料能力更精準。

| 等級 | 定義 | 適用區塊 |
|:---|:---|:---|
| 精準資料 | 直接來自日 K、即時報價、法人/融資融券原始資料或可重現公式 | MA、KD、MACD、成交量、近 N 日法人買賣超、融資融券餘額 |
| 規則估算 | 使用明確 rule-based 公式推導，非交易所原始欄位 | 支撐壓力、短線綜合分數、W 底/M 頭、量價結構 |
| 資料降級 | 缺少 tick、券商分點或真正內外盤資料時，以替代資料近似 | 主力出貨警示、籌碼集中度、買賣力道估算 |
| AI 文字解讀 | AI 只根據已提供資料轉成中文分析，不得自行編造數字 | 公司/產業概況、隔日操作劇本、整體結論 |
| 資料不足 | 資料列數不足、API 失敗或 local cache 不完整 | 對應區塊顯示「資料不足」或降級提示 |

### 2.1 Phase 8-A：技術面自動判讀引擎

> 把指標數值轉成人可讀的文字結論

**目標**：給定一檔股票的日 K 資料，自動產出截圖「區塊 3 技術分析總覽」、「區塊 7 短線綜合分數」、「區塊 8 關鍵價位」、「區塊 9 量價結構分析」等文字化結論。

**新增模組**：`src/analysis/technical_summary.py`

**輸入**：`pd.DataFrame`（日 K raw OHLCV；可選附 adjusted OHLCV 供報酬/長期比較使用）

價格尺度規則：
- K 線圖、即時價、支撐壓力、操作劇本價位一律使用 raw price，確保與盤中報價同尺度。
- adjusted price 僅供長期報酬、除權息調整後趨勢比較使用，不得混入即時價位判讀。

**輸出**：`TechnicalSummary` dataclass，結構如下：

```python
@dataclass
class PriceLevel:
    value: float                 # 價位
    label: str                   # "近期高點" / "前高" / "MA20" / "MA60" / ...
    kind: str                    # "resistance" / "support"


@dataclass
class TechnicalSummary:
    # 區塊 3：技術分析總覽
    trend_direction: str          # "多頭趨勢" / "空頭趨勢" / "盤整"
    ma_status: str                # "多頭排列 (5>20>60)" / "空頭排列" / "糾結"
    kd_status: str                # "KD 高檔鈍化" / "KD 低檔鈍化" / "KD 黃金交叉" / ...
    macd_status: str              # "正值收斂" / "負值擴張" / "零軸上方" / ...
    volume_status: str            # "量能放大" / "量能略縮" / "爆量"
    volume_price_relation: str    # "價漲量增" / "價漲量縮（短線整理）" / ...

    # 區塊 7：短線綜合分數
    short_term_score: float       # 0.0~1.0，綜合 MA、KD、量價、突破狀態
    short_term_label: str         # "強勢偏多" / "中等偏多" / "中性" / "偏空"
    short_term_components: dict[str, float]  # {"ma": 0.8, "kd": 0.6, "volume_price": 0.7, "breakout": 0.5}

    # 區塊 8：關鍵價位
    resistance_levels: list[PriceLevel]  # 壓力區（近期高點 + 前高）
    support_levels: list[PriceLevel]     # 支撐區（近期低點 + MA 均線）

    # 區塊 9：量價結構
    volume_price_divergence: str  # "量價背離：價漲量縮" / "量價同步" / ...
    ma_bias: str                  # "與 MA20 乖離約 +4.61%，偏高"
    chip_behavior: str            # 需 8-C 籌碼資料注入，否則留空
    operation_observation: str    # 綜合觀察文字
```

**判讀規則（rule-based）**：

| 指標 | 判讀邏輯 |
|:---|:---|
| 趨勢方向 | MA5 > MA20 > MA60 → 多頭；反之空頭；交錯 → 盤整 |
| MA 狀態 | 三線排列順序 + 最新 close 相對位置 |
| KD 狀態 | K > 80 → 高檔鈍化；K < 20 → 低檔鈍化；K 上穿 D → 黃金交叉 |
| MACD 狀態 | DIF > 0 且 DIF > DEA → 正值擴張；DIF > 0 且 DIF < DEA → 正值收斂 |
| 量能狀態 | 今日量 vs 5 日均量：> 1.5x 放大、0.7x~1.5x 正常、< 0.7x 略縮 |
| 短線綜合分數 | MA 結構、KD 位置/交叉、量價關係、突破狀態四項加權，輸出 0~100% |
| 壓力區 | 近 60 日最高價、近 20 日最高價（取不重複的 2 個），保留來源 label |
| 支撐區 | 近期低點、MA20、MA60（取最近的 2~3 個），保留來源 label |
| 乖離率 | `(close - MA20) / MA20 × 100%` |

短線綜合分數建議權重：

| 組成 | 權重 | 評分重點 |
|:---|:---|:---|
| MA 結構 | 30% | MA5 > MA20 > MA60、close 在均線上方得分較高 |
| KD 狀態 | 25% | 黃金交叉、K/D 位階、是否高檔鈍化 |
| 量價關係 | 25% | 價漲量增、價跌量縮較佳；價漲量縮降分 |
| 突破狀態 | 20% | 是否突破近 20/60 日高點，或跌破關鍵支撐 |

UI 可顯示為圓弧儀表，但文字需標示「短線綜合分數」，不要稱為統計勝率，除非僅使用歷史上漲日比例。

**不使用 AI**：純 rule-based template engine，`ai.enabled=false` 時照常運作。

**測試**：
- `tests/test_technical_summary.py`
- 手工構造 fixture（多頭排列、空頭排列、盤整各一組），assert 每個欄位文字正確
- 邊界：全部 NaN（資料不足）、僅 5 根 bar（MA60 算不出來）

**AI 串接**：否
**新增管線**：無
**預估量體**：小

#### 2.2.1 K 線型態表

利用 pandas-ta 的 `cdl_pattern(name=...)` 與自定規則辨識當日 K 線型態。

技術確認結果（2026-05-10）：
- `cdl_pattern` ✅ 存在，接受 `name` 參數（字串或 list）
- `cdl_doji` ✅ 存在（獨立函式）
- `CDL_PATTERN_NAMES` 共 62 種型態，含 `hammer`、`hangingman`、`engulfing`、`morningstar`、`eveningstar` 等全部需要的型態

| 型態 | 實作方式 | 判讀 |
|:---|:---|:---|
| 長紅 K / 長黑 K | 自定（body > 2× avg body） | 多方 / 空方力道 |
| 十字線 | `cdl_doji(open, high, low, close)` | 多空拉鋸 |
| 錘子 / 吊人 | `cdl_pattern(open, high, low, close, name=["hammer", "hangingman"])` | 反轉訊號 |
| 吞噬 | `cdl_pattern(..., name="engulfing")` | 反轉確認 |
| 晨星 / 夜星 | `cdl_pattern(..., name=["morningstar", "eveningstar"])` | 強反轉 |
| 帶上影線 / 下影線 | 自定（shadow > 2× body） | 上檔壓力 / 下檔支撐 |

輸出 `list[CandlePattern]`，每筆含 `name: str, detected: bool, description: str`。

#### 2.2.2 圖表型態偵測

| 型態 | 偵測演算法 |
|:---|:---|
| W 底（雙底） | 近 N 日找 2 個相近低點（誤差 < 3%），中間有反彈高點 |
| M 頭（雙頂） | 近 N 日找 2 個相近高點（誤差 < 3%），中間有回落低點 |
| 頭肩（選配） | 三峰結構，中間最高，兩側對稱（複雜度高，列為選配） |

輸出 `PatternResult`，含 `pattern_type: str, formed: bool, description: str`。

偵測不到時明確標示「未形成標準 X 型態」，不硬套（與截圖風格一致）。

#### 2.3.4 籌碼分析模組

```python
@dataclass
class ChipSummary:

    foreign_net_n_days: int       # 外資近 N 日淨買賣（張）
```

### 2.4 Phase 8-D：即時行情接入

> TWSE MIS 公開 API 即時報價

**目標**：盤中取得即時成交價、漲跌、成交量、五檔報價，產出截圖「區塊 1 頂部行情條」與「區塊 10 買賣力道估算」。

**新增模組**：`src/data/realtime.py`

#### 2.4.1 API 端點

技術確認結果（2026-05-10）：**只需一個端點**。`mis.twse.com.tw` 同時支援上市（`tse_`）和上櫃（`otc_`），用 `ex_ch` 參數的 prefix 區分。TPEx 的 `mis.tpex.com.tw` 回傳 HTML 而非 JSON，不可用，因此不列為 Phase 8 即時報價資料源。

| 市場 | `ex_ch` 格式 | 驗證結果 |
|:---|:---|:---|
| 上市（tse） | `tse_{symbol}.tw` | ✅ 2330 成功 |
| 上市（tse） | `tse_3583.tw` | ✅ 3583 辛耘成功（FinMind 分類為 twse） |
| 上櫃（otc） | `otc_{symbol}.tw` | ✅ 6547 高端成功 |

統一端點：`https://mis.twse.com.tw/stock/api/getStockInfo.jsp?ex_ch={prefix}_{symbol}.tw`

**SSL 注意**：TWSE 憑證缺 Subject Key Identifier，Python 3.13 嚴格檢查會觸發 `SSLCertVerificationError`。可對 `mis.twse.com.tw` 這個單一 host 設定 `verify=False` 並在該請求範圍內抑制 `InsecureRequestWarning`。不得全域關閉 TLS 驗證或全域 suppress warning。

#### 2.4.3 輸出 dataclass

```python
@dataclass
class RealtimeQuote:
    symbol: str
    name: str               # 股票名稱
    price: float            # 成交價
    change: float           # 漲跌（price - yesterday_close，本地計算）
    change_pct: float       # 漲跌幅 %（本地計算）

    high: float
    low: float
    yesterday_close: float  # 昨收

    best_ask: list[float]   # 五檔賣價（由低到高）
    best_bid_vol: list[int] # 五檔買量
    best_ask_vol: list[int] # 五檔賣量
    is_market_open: bool    # 目前是否盤中
```

TWSE API 原始欄位映射（技術確認 2026-05-10）：

| API key | 意義 | 對應欄位 | 解析方式 |
|:---|:---|:---|:---|
| `c` | 股票代碼 | `symbol` | 直接取 |
| `n` | 股票名稱 | `name` | 直接取（Big5 編碼，requests 自動解碼） |
| `z` | 成交價 | `price` | `float(z)`，`"-"` 時為無成交 |
| `y` | 昨收 | `yesterday_close` | `float(y)` |
| `o` | 開盤 | `open` | `float(o)` |
| `h` | 最高 | `high` | `float(h)` |
| `l` | 最低 | `low` | `float(l)` |
| `v` | 累積成交量（張） | `volume` | `int(v)` |
| `t` | 最後成交時間 | `timestamp` | `"HH:MM:SS"` 格式 |
| `b` | 五檔買價 | `best_bid` | `"_"` 分隔，filter 空字串，`float` 轉換 |
| `a` | 五檔賣價 | `best_ask` | 同上 |
| `g` | 五檔買量 | `best_bid_vol` | `"_"` 分隔，filter 空字串，`int` 轉換 |
| `f` | 五檔賣量 | `best_ask_vol` | 同上 |
| `u` | 漲停價 | （不映射，可選保留） | — |
| `w` | 跌停價 | （不映射，可選保留） | — |
| `ex` | 市場 | （用於驗證路由） | `"tse"` / `"otc"` |

`change` 和 `change_pct` 由本地計算：`change = price - yesterday_close`，`change_pct = change / yesterday_close * 100`。

#### 2.4.4 盤後 fallback

盤後（13:30 後 / 假日）呼叫 `fetch_quote` 時：
- TWSE MIS API 仍會回傳最後收盤資料
- `is_market_open = False`
- UI 顯示「盤後資料」標籤

#### 2.4.5 快取策略

- 同一 symbol 在 `_cache_ttl` 秒內重複查詢，直接回 cache
- 避免頻繁打 API 被封鎖
- 預設 10 秒，可由 `config.yaml` 覆寫

#### 2.4.6 買賣力道估算（區塊 10）

TWSE MIS API 回傳五檔報價但不直接提供內外盤統計。Phase 8 僅做買賣力道估算，近似方案：

- **盤中**：用五檔買賣量的總量比作為多空力道估算
- **盤後**：用 `(close - open) / (high - low)` 作為日內多空力道近似指標


**注意事項**：
