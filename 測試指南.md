驗證：

```bash
uv run python -c "import pandas, duckdb, pyarrow, plotly, yaml; print('ALL OK')"

```

### Step 3：設定 .env

```bash
# 複製範本
copy .env.example .env
```

編輯 `.env`，填入：

```
FINMIND_TOKEN=你的_FinMind_Token
ANTHROPIC_API_KEY=sk-ant-你的_API_Key
OPENAI_API_KEY=sk-你的_OpenAI_API_Key
GEMINI_API_KEY=你的_Gemini_API_Key
```

> **沒有 FINMIND_TOKEN / AI API Key？**
> 單元測試不需要。整合測試（標記 `@pytest.mark.integration`）才需要。
> 先跳過 `.env`，只跑單元測試也可以推進開發。

### Step 4：設定 pytest marker

確認 `pyproject.toml` 包含：

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
pythonpath = ["."]
markers = [
    "integration: 需要外部 API 或網路連線的測試",
]
```

這樣 `@pytest.mark.integration` 才不會出現 `PytestUnknownMarkWarning`。

| 問題 | 解法 |
|------|------|
| `ModuleNotFoundError: No module named 'src'` | 確認 `pyproject.toml` 有 `pythonpath = ["."]`；確認從專案根目錄執行 pytest |
| `requests.exceptions.ConnectionError` | 檢查網路；FinMind API 是否在維護 |
| `FINMIND_TOKEN not found` | 檢查 `.env` 是否存在、token 值是否正確、`config.py` 是否呼叫 `load_dotenv()` |
| yfinance 回傳空 DataFrame | yfinance 對台股偶爾不穩定，重試幾次；確認 symbol 格式為 `"2330.TW"` |

| `test_duckdb_meta_upsert_overwrite` | 同一 `(symbol, freq)` upsert 兩次 → 後者覆蓋前者 | INSERT 未處理衝突（需 `ON CONFLICT ... DO UPDATE`） |

#### Review 補充待加入測試（2026-04-26）

| 測試名稱 | 驗證什麼 | 為何需要 |
|----------|---------|----------|
| `test_storage_rejects_path_traversal_symbol` | `symbol` 含 `..`、路徑分隔符或絕對路徑時必須拒絕；合法 symbol 的輸出路徑 `resolve()` 後仍在 `data_dir` 內 | 規格要求資料只能落在 `data/raw/tw/{symbol}` / `data/processed/tw/{symbol}`；避免外部輸入造成任意路徑寫入 |
| `test_storage_uses_default_tw_market_path` | 未指定 market 時仍寫入 `data/raw/tw/{symbol}`；指定 `market="tw"` 行為相同 | 先保留未來美股擴充接口，但不能破壞現有台股預設路徑 |
| `test_storage_rejects_unknown_market` | 傳入尚未支援的 market，例如 `us`，應明確拒絕或標示未支援 | 本階段不實作美股；避免使用者誤以為美股已可用 |

#### 故障排除

| 問題 | 解法 |
|------|------|
| `pyarrow.lib.ArrowInvalid: Casting from timestamp[ns] to timestamp[ns, tz=Asia/Taipei]` | 確認寫入前 DataFrame 的 `date` 已是 tz-aware |
| DuckDB `Catalog Error: Table 'data_meta' does not exist` | `__init__` 中要執行 `CREATE TABLE IF NOT EXISTS` |
| `assert_frame_equal` 失敗但肉眼看數值一樣 | 通常是 dtype 不同（int64 vs Int64）或時區不同 |
